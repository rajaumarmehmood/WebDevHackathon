\section{Executive Summary}

CareerAI is an intelligent web application designed to help students and job seekers navigate their career journey through AI-powered resume analysis, automated job discovery, and personalized interview preparation. The platform leverages modern web technologies and artificial intelligence to provide a comprehensive job search and interview preparation solution.

\subsection{Key Features}
\begin{itemize}[leftmargin=*]
    \item \textbf{Smart Resume Analysis}: PDF upload with AI-powered extraction of skills, experience, and qualifications
    \item \textbf{Automated Job Discovery}: Web scraping and API integration to find relevant job opportunities
    \item \textbf{AI-Powered Job Matching}: Intelligent ranking of jobs based on user profile compatibility
    \item \textbf{Interview Preparation}: Research-backed, personalized interview questions and study guides
    \item \textbf{Career Analytics Dashboard}: Visual insights into job search progress and skill gaps
    \item \textbf{Video Interview Practice}: AI-powered mock interview system
\end{itemize}

\section{Database Architecture}

\subsection{Database Schema}

The application uses Supabase (PostgreSQL) with the following tables:

\subsubsection{Profiles Table}
Extends Supabase auth.users with additional user information.
\begin{lstlisting}[language=SQL]
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  email TEXT NOT NULL,
  name TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
\end{lstlisting}

\subsubsection{Resumes Table}
Stores uploaded resume files and AI analysis results.
\begin{lstlisting}[language=SQL]
CREATE TABLE resumes (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES auth.users(id) UNIQUE,
  file_name TEXT NOT NULL,
  file_url TEXT,
  analysis JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
\end{lstlisting}

\subsubsection{Job Applications Table}
Tracks discovered jobs and their match scores.
\begin{lstlisting}[language=SQL]
CREATE TABLE job_applications (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  job_title TEXT NOT NULL,
  company TEXT NOT NULL,
  location TEXT NOT NULL,
  salary_range TEXT,
  match_score INTEGER DEFAULT 0,
  status TEXT DEFAULT 'saved',
  applied_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
\end{lstlisting}

\subsubsection{Interview Preps Table}
Stores AI-generated interview preparation materials.
\begin{lstlisting}[language=SQL]
CREATE TABLE interview_preps (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  company TEXT NOT NULL,
  role TEXT NOT NULL,
  technologies TEXT[] NOT NULL DEFAULT '{}',
  prep_material JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
\end{lstlisting}

\subsection{Row Level Security (RLS)}

All tables implement Row Level Security policies ensuring users can only access their own data:
\begin{itemize}[leftmargin=*]
    \item SELECT: Users can view only their own records
    \item INSERT: Users can create records only for themselves
    \item UPDATE: Users can modify only their own records
    \item DELETE: Users can delete only their own records
\end{itemize}

\subsection{Database Triggers}

\begin{itemize}[leftmargin=*]
    \item \textbf{Auto Profile Creation}: Automatically creates a profile when a new user signs up
    \item \textbf{Updated At Timestamp}: Automatically updates the updated\_at field on record modifications
\end{itemize}

\subsection{Resume APIs}
\begin{itemize}[leftmargin=*]
    \item \texttt{POST /api/resume/upload} - Upload and analyze resume PDF
    \item \texttt{GET /api/resume/upload?userId=<id>} - Retrieve user's resume data

\textbf{Upload Process:}
\begin{enumerate}
    \item Validate PDF file (max 10MB)
    \item Extract text using pdf-parse library
    \item Clean and structure extracted text
    \item Analyze with Google Gemini AI
    \item Upload PDF to Supabase Storage
    \item Save analysis to database
    \item Trigger background job discovery
\end{enumerate}

\subsection{Job Discovery APIs}
\begin{itemize}[leftmargin=*]
    \item \texttt{POST /api/jobs/discover} - Search and match jobs
    \item \texttt{GET /api/jobs/discover?userId=<id>} - Get saved job matches
\end{itemize}

\textbf{Discovery Process:}
\begin{enumerate}
    \item Retrieve user's resume analysis
    \item Search jobs via SERP API (Google Jobs)
    \item Filter jobs with valid descriptions
    \item Match jobs to user profile using AI
    \item Calculate match scores and reasons
    \item Save matches to database
    \item Update user analytics
\end{enumerate}

\subsection{Interview Preparation APIs}
\begin{itemize}[leftmargin=*]
    \item \texttt{POST /api/interview-prep/generate} - Generate interview prep materials
    \item \texttt{GET /api/interview-prep?userId=<id>} - Get all interview preps
    \item \texttt{GET /api/interview-prep/<id>} - Get specific interview prep
\end{itemize}

\subsection{Analytics APIs}
\begin{itemize}[leftmargin=*]
    \item \texttt{GET /api/analytics?userId=<id>} - Get user analytics and insights
    \item \texttt{GET /api/dashboard/stats?userId=<id>} - Get dashboard statistics
\end{itemize}

\section{Core Features Implementation}

\subsection{Intelligent Resume Analysis}

\subsubsection{PDF Processing}
The system uses the \texttt{pdf-parse} library to extract text from uploaded PDF resumes:

\begin{lstlisting}[language=JavaScript]
export async function extractTextFromPDF(buffer: Buffer) {
  const PDFParse = getPDFParseClass();
  const parser = new PDFParse({ data: buffer });
  const result = await parser.getText();
  return result.text;
}
\end{lstlisting}

\subsubsection{Text Cleaning}
Extracted text is cleaned to remove excessive whitespace and special characters while preserving resume structure:
\begin{itemize}[leftmargin=*]
    \item Normalize whitespace and line breaks
    \item Remove invalid characters
    \item Preserve important punctuation (@, ., -, etc.)
    \item Validate minimum content length
\end{itemize}

\subsubsection{AI Analysis}
Google Gemini AI analyzes the cleaned text to extract:
\begin{itemize}[leftmargin=*]
    \item Technical skills and proficiency levels
    \item Years of experience
    \item Education background
    \item Project experience
    \item Certifications
    \item Career level (Entry, Mid, Senior)
\end{itemize}

\subsection{Automated Job Discovery}

\subsubsection{Job Search Integration}
The platform integrates with SERP API to access Google Jobs:

\begin{lstlisting}[language=JavaScript]
export async function searchJobs(
  skills: string[],
  location: string = 'Remote',
  limit: number = 20
): Promise<Job[]>
\end{lstlisting}

\textbf{Search Parameters:}
\begin{itemize}[leftmargin=*]
    \item Query: Top 3 user skills + location
    \item Engine: Google Jobs
    \item Location: Pakistan, Remote, or custom
    \item Limit: Configurable (default 20)
\end{itemize}

\subsubsection{Job Filtering}
Jobs are filtered to ensure quality:
\begin{itemize}[leftmargin=*]
    \item Minimum description length (50 characters)
    \item Valid company and title information
    \item Relevant technical requirements
\end{itemize}

\subsubsection{Requirement Extraction}
The system extracts technical requirements from job descriptions by matching against:
\begin{itemize}[leftmargin=*]
    \item Common tech skills database (JavaScript, Python, React, etc.)
    \item User's specific skills
    \item Framework and tool mentions
\end{itemize}

\subsection{AI-Powered Job Matching}

\subsubsection{Match Score Calculation}
Google Gemini AI analyzes each job against the user profile to generate:
\begin{itemize}[leftmargin=*]
    \item \textbf{Match Score}: 0-100 percentage indicating compatibility
    \item \textbf{Match Reasons}: Specific explanations for the score
    \item \textbf{Skill Alignment}: Which user skills match job requirements
    \item \textbf{Experience Fit}: Whether experience level is appropriate
\end{itemize}

\subsubsection{Ranking Algorithm}
Jobs are sorted by:
\begin{enumerate}
    \item Primary: Match score (descending)
    \item Secondary: Posted date (recent first)
    \item Tertiary: Salary range (if available)
\end{enumerate}

\subsection{Interview Preparation System}

\subsubsection{Multi-Step Research Process}
When a user requests interview prep, the AI:
\begin{enumerate}
    \item Searches for current information about the specified technologies
    \item Identifies trending topics and best practices
    \item Analyzes common interview patterns for the role
    \item Filters outdated or irrelevant information
    \item Generates difficulty-appropriate questions
\end{enumerate}

\subsubsection{Generated Materials}
\begin{itemize}[leftmargin=*]
    \item \textbf{Technical Questions}: Technology-specific (e.g., Node.js event loop)

    \item \textbf{Coding Challenges}: Appropriate to experience level
    \item \textbf{Study Guide}: Key concepts to review
    \item \textbf{Company Research}: Specific company insights
\end{itemize}

\section{Authentication \& Security}

\subsection{Authentication Flow}

\subsubsection{Sign Up Process}
\begin{enumerate}
    \item User submits email, password, and name
    \item Supabase creates auth user
    \item Database trigger creates profile record
    \item Confirmation email sent
    \item User redirects to email confirmation page
\end{enumerate}

\subsubsection{Login Process}
\begin{enumerate}
    \item User submits credentials
    \item Supabase validates and issues JWT
    \item Token stored in HTTP-only cookie
    \item User redirects to dashboard
    \item Middleware validates session on protected routes
\end{enumerate}

\subsection{Security Measures}

\begin{itemize}[leftmargin=*]
    \item \textbf{Row Level Security}: Database-level access control
    \item \textbf{JWT Tokens}: Secure session management
    \item \textbf{HTTP-Only Cookies}: XSS protection
    \item \textbf{HTTPS}: Encrypted data transmission
    \item \textbf{Input Validation}: Server-side validation for all inputs
    \item \textbf{File Upload Limits}: 10MB max for PDFs
    \item \textbf{CORS Configuration}: Restricted API access
    \item \textbf{Environment Variables}: Sensitive data protection
\end{itemize}

\section{Data Flow}

\subsection{Resume Upload Flow}
\begin{enumerate}
    \item User uploads PDF via drag-and-drop interface
    \item Frontend validates file type and size
    \item File sent to \texttt{/api/resume/upload}
    \item Server extracts text from PDF
    \item Text cleaned and structured
    \item AI analyzes resume content
    \item PDF uploaded to Supabase Storage
    \item Analysis saved to database
    \item Response returned to user immediately
    \item Background job search initiated
    \item Jobs matched and saved asynchronously
\end{enumerate}

\subsection{Job Discovery Flow}
\begin{enumerate}
    \item System retrieves user's resume analysis
    \item Constructs search query from top skills
    \item Calls SERP API for Google Jobs
    \item Filters jobs with valid descriptions
    \item Extracts requirements from descriptions
    \item AI matches jobs to user profile
    \item Calculates match scores and reasons
    \item Saves matches to database
    \item Updates user analytics
    \item Returns sorted job list to frontend
\end{enumerate}

\subsection{Interview Prep Flow}
\begin{enumerate}
    \item User inputs company, role, and technologies
    \item Request sent to \texttt{/api/interview-prep/generate}
    \item AI performs multi-step research
    \item Generates personalized questions
    \item Creates study guide
    \item Saves prep materials to database
    \item Returns formatted content to user
    \item User can download or view online
\end{enumerate}
